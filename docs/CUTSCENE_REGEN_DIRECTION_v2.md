## 1. LAWS (every generation)
- NO TEXT IN IMAGE, ever. No glyphs, no script (carvings/objects/
  characters — the script veto). Free sky notation exempt per ruling.
- Shiva: silhouette with blue-throat glow ONLY, never a face (ch5
  optional beat). Mohini: slender feminine silhouette ONLY (c7_v1).
- Gore law: aftermath not violence; no blood.
- SAFE MARGIN: nothing critical in the outer 10% (Ken Burns 1.05).
- Two versions per panel:
  WIDE 3:2 landscape (desktop) and MOBILE 2:3 portrait (phone).
  Recompose per orientation — wide spreads laterally, mobile stacks
  vertically. Same subject, same palette, same moment.
- Chapter color story: ch1 dusk-gold, ch2 dawn-ivory, ch3 gold-on-
  crimson, ch4 ocean-silver, ch5 poison black-green, ch6 radiant
  gold, ch7 storm steel.

## 2. STYLE STEM (prepend to every prompt)
> epic painterly Indian mythology fantasy, rich oil-painting texture,
> dramatic light, cinematic composition, no text, no lettering

## 3. CHARACTER DESCRIPTOR BLOCKS (paste verbatim wherever named)

## 5. DELIVERY AND REPLACEMENT
- Filenames: MOBILE keeps the shipped name exactly —
  b1c{ch}_{slot}.jpg — a drop-in byte-swap (the ?v= stamp bumps in
  the same commit, per law). WIDE adds the suffix —
  b1c{ch}_{slot}_wide.jpg — wired by task T56 (media-query swap at
  1024px on the proven per-surface pattern; authored when the first
  wide batch is intaken).
- Send picks in chapter batches. Claude audits (per-panel veto
  order: no text/glyphs, composition survives the motion, palette
  on the chapter arc, safe margin), compresses to ~200-280 KB JPEG,
  renames, and returns the intake block.
- Panel count: 27 panels, 54 images. Double-duty panels (c3_i1,
  c7_i2) are one image each per orientation, reused by the engine.
