# DIVYA YUDDHA — CARD FRAME PROGRAM v1
# The GPT master-template frame system, canonized from the ten Deva
# exports of 2026-07-17. This doc turns precedent into law so the
# remaining ~170 frames inherit rules, not vibes.
# Authority order: engine (rules text) → DEVA_FACTION_CARDS_v1.md
# pattern (per-faction compendium) → this doc (frame system).

═══════════════════════════════════════════
## 1. PIPELINE LAW (non-negotiable)
═══════════════════════════════════════════
1. TEXT SOURCE: every rules line and flavor line on a frame comes
   from that faction's compendium doc — which is itself built ONLY
   from an engine pull (abilities, verbatim) and the canonical
   flavor record. The compendium is the ONLY permitted text feed
   to generation. CardRoster v1.0 and memory are forbidden sources
   (the Indra's Net rule, at scale).
2. PER-FRAME AUDIT before a frame is locked:
   a. TXT verbatim vs the compendium (word-for-word; keyword may
      be split into the header per §4).
   b. Flavor verbatim vs the compendium.
   c. Name + epithet per the compendium.
   d. Gem color = the card's rarity rung (§3).
   e. Faction crest + roundel ground = the card's faction (§2).
   f. SCRIPT AUDIT at full res: any mandala rings, halos, or
      ornament bands carrying character-like marks must resolve
      into real, deliberate Devanagari or be edited to pure
      geometry. Glyph-soup is a rejection. (Standing holds:
      Vajra, Brahmastra — rings pending Sangbaran's full-res eye.)
   g. Register: reverence gate for deities; rank-appropriate
      energy (Commons hold still; see Deva Soldier v2 ruling).
3. A frame failing any item gets a text re-export or region-edit,
   never a hand-waved pass. Art re-rolls only for register/
   composition failures.

═══════════════════════════════════════════
## 2. THE TWO-AXIS SYSTEM
═══════════════════════════════════════════
FACTION IS CONSTANT per card: the crest (top-left, on the faction
ground) and the power roundel's ground carry faction identity.
RARITY IS VARIABLE: the gems (title-band finial, type-band
diamonds, bottom finial, rarity-plaque stars) and the rarity
plaque carry the rung color.
The two axes never swap carriers. (Watch-item from Ashwini: rarity
color must never bleed into the crest ground.)

FACTION COLORWAYS (grounds; crest per assets/img/crest_*.png):
- DEVA: deep navy ground, gold metal (shipped standard).
- ASURA: crimson-black ground, gold metal with ember warmth.
- VANARA: forest-black ground, gold metal.
- NAGA: teal-black ground, gold-verdigris metal permitted.
Frame metal stays the gold family for all four — faction lives in
the grounds and crest, not in recoloring the filigree.

## 4. TEMPLATE CONVENTIONS
═══════════════════════════════════════════
- NAME BAND: name large + epithet small caps beneath, both from
  the compendium.
- KEYWORD HEADER: the engine keyword (ON PLAY / PASSIVE /
  TRIGGERED / ROUND END) split as a header when present; EFFECT
  when the engine text carries none (astras, mantras).
- RARITY: top-right navy plaque, ONE instance; type band center,
  ONE instance. No bottom roundel, no inline "RARITY:" line
  (the redundancy trims, ruled on the Indra master).
- POWER: roundel with the number; N/A for powerless types.
- FLAVOR: unquoted italic (the master's convention — applies to
  every card on this template; never mixed with quoted).
- ORNAMENT-BY-TYPE (candidate convention, confirm as types
  export): lotus = Unit; star = Astra; Mantra/Artifact/Hero
  ornaments set by their first accepted export.
- FACTION LABEL: "DEVA FACTION" wording currently shipped on all
  ten; standing one-liner to shorten to the faction name alone
  remains open — whatever the Asura master carries decides it.
- CHARIOT AMENDMENT: the no-chariots law is MJ-scoped only; this
  pipeline renders chariots subject to normal audit (repealed by
  the Surya Dev export).

## 5. SCOPE — THE STANDING DECISION
═══════════════════════════════════════════
Ten Deva frames exist; the run is a full program in practice.
The unruled question is DEPLOYMENT:
(a) RATNA-FIRST: the 24 Ratna cards adopt this frame in the Vault
    (the tier that sells "luxury made visible"); the game-wide
    migration decides after.
(b) FULL MIGRATION: all 176 + token re-frame on this template —
    a program with per-faction compendium builds (Asura: this
    session's docs; Naga: direction doc on disk; Vanara: not yet
    written), four colorway masters, and an asset-pipeline pass
    (export → audit → assets/cards → make_thumbs).
Until ruled, exports continue under §1 audit and nothing ships
into assets/cards — the shipped Canva frames remain live.
